# solutionDevelopment2

### 【前提・システム仕様】

- **提供形態:** アプリケーション（ソフトウェア）として構築する
- **データ保存先:** フォーマットごとの枠座標・タグ情報はJSONファイル（`formats.json` 等）に保存する
- **出力形式:** 抽出結果のExcelファイルは既存ファイルへの追記ではなく、「新規作成」で出力する

---
### 【一連の動作フロー】

#### 0. 初期画面（モード選択）
1. **「フォーマット登録機能」** または **「文字認識機能」** のどちらかを選択する。

#### A. フォーマット登録機能（マスターデータ作成）
1. **アップロード:** ユーザーが空のフォーマットPDF（雛形）をアップロードする。
2. **枠線認識:** システムがPDF内の枠線を自動で認識する。
3. **基準点記録（※保留）:** フォーマット判別用の指標（一番左上の枠など）の位置を記録する。
4. **重複チェック（※方法未定）:** 既に登録されているフォーマットかどうかを判別する。
    - **登録済みの場合:** エラーを表示し、初期画面に戻る。
    - **未登録の場合:** 次の手順へ進む。
5. **UI表示:** 認識した枠線をハイライトした画像を画面に表示する。
6. **タグ付け（座標指定）:** ユーザーに抽出したい枠を選択させ、その枠に書かれる「情報の種類（タグ名：例『依頼日』『担当者』など）」を入力させる。
7. **保存:** すべてのタグ付けが終わったら登録を実行し、座標とタグの情報をJSONファイルに保存する。

#### B. 文字認識機能（メイン機能）
1. **アップロード:** ユーザーがデータ記入済みのPDFをアップロードする。
2. **枠線認識:** システムがPDF内の枠線を認識する。
3. **フォーマット判別（※方法保留）:** 登録済みのフォーマットと照合・判別する。
    - **登録されていない場合:** エラーを表示し、初期画面に戻る。
    - **登録されている場合:** 次の手順へ進む。
4. **PDF種別判別とデータ抽出:** アップロードされたPDFが「テキストベース」か「画像ベース」かを自動判別し、抽出処理を分岐させる。
    - **テキストベースの場合:** Pythonのライブラリ（`pdfplumber` 等）を使用し、登録した座標から直接テキストデータを抽出・取得する。
    - **画像ベースの場合:** 登録された座標部分の画像を切り取り（クロップ）、OCRエンジンにかけてテキストを抽出・取得する。
5. **結果表示:** 取得した文字列と、タグ付けした情報の種類をUI画面に一覧表示する。（※左側にPDFのイメージ画像、右側に抽出結果などを配置）
6. **手動修正:** ユーザーが表示されている情報を手動で修正する。
    - 「編集」ボタンを押下すると、表示されている文字列のテキストボックスが編集可能になる。
    - 修正後、「保存」ボタンを押下して修正内容をシステムに反映させる。
7. **次のアクション選択:** 続けて処理を行うか、データを出力するかを選択する。
    - **追加でPDFをアップロードする場合:** 文字認識機能の最初（手順1）に戻る。
    - **Excelをダウンロードする場合:** 取得・修正した文字列を、タグ付けした情報の種類（ヘッダー）をもとにExcelに転記し、「新規のExcelファイル」としてダウンロードさせる。
